# ERP-Web 需求设计文档(MVP 第一版)

> 日期:2026-06-29
> 状态:已与用户确认范围,待写实现计划

## 1. 项目定位

中小企业 ERP 的 **Web 前端**。后端为独立服务(并行开发中),前端通过 REST API 对接。

## 2. 范围(第一版)

只做两大模块,跑通主流程:

- **系统管理(权限)** —— 地基,所有页面的前提
- **供应链(进销存)** —— 业务核心

**明确不做(留接口、按需再加)**:财务管理、生产控制。

## 3. 技术栈

| 类别 | 选型 |
|---|---|
| 框架 | Vue 3 |
| 构建 | Vite |
| 语言 | TypeScript |
| 状态 | Pinia |
| 路由 | Vue Router |
| HTTP | axios |
| UI 组件 | Ant Design Vue |
| 数据来源 | Mock(后端未就绪),约定接口后切真实地址 |

## 4. 后端对接策略

后端并行开发、暂无接口 → **先约定接口规范 + Mock 层**:

- 前后端先约定一套 REST 接口约定(路径、请求/响应结构、分页、错误码)。
- 前端用 Mock 数据(Vite mock 插件或 MSW)把页面跑起来。
- axios 统一封装:baseURL 走环境变量,token 注入请求头,统一错误/loading 处理。
- 后端就绪后,改环境变量切真实地址,关掉 mock 即可,页面代码不动。

## 5. 功能清单

### 5.1 系统管理(权限)

- 登录 / 登出(token 鉴权)
- 用户管理:增删改查、启用停用、重置密码
- 角色管理:增删改查、给角色分配权限
- 菜单 / 权限:菜单路由 + 按钮级权限点
- 组织 / 部门:树形部门结构(简版)
- 个人中心:改密码、查看个人信息

### 5.2 供应链(进销存)

**基础资料**
- 商品 / 物料:增删改查、分类、单位、规格
- 供应商管理
- 客户管理
- 仓库管理

**采购**
- 采购订单:下单、审核、收货
- 采购入库单

**销售**
- 销售订单:下单、审核、发货
- 销售出库单

**库存**
- 实时库存查询(按仓库 / 商品)
- 出入库流水
- 库存盘点(简版)

## 6. UI 方案

内部后台,**采用 Ant Design Vue 默认设计体系**,不做定制化视觉(对内部 ERP 而言定制视觉属于过度设计)。重点是布局结构与交互一致性。

### 6.1 整体布局(经典中后台)

```
┌─────────────────────────────────────┐
│  Logo   顶栏:面包屑 · 用户 · 退出      │
├──────────┬──────────────────────────┤
│ 左侧菜单  │   内容区                  │
│ (按权限   │   列表页 / 表单页          │
│  动态生成)│                          │
└──────────┴──────────────────────────┘
```

### 6.2 统一交互范式(所有模块复用)

- **列表页**:顶部筛选条件 → AntD Table(排序/分页)→ 行内操作(编辑/删除)→ 右上角"新增"
- **新增 / 编辑**:Modal 弹窗或 Drawer 抽屉表单,字段用 AntD Form 校验
- **删除**:Popconfirm 二次确认
- **全局**:loading、消息提示(message)、空状态统一封装一次,各页复用

### 6.3 主题

AntD 默认蓝色主题。换主题色为 AntD 配置项级改动,无需重构。

## 7. 落地顺序

每块单独一轮 spec → plan → 实现,逐块交付:

```
① 系统管理(权限)   ← 先做,提供登录 + 框架 + 权限基座
② 供应链(进销存)   ← 复用①的权限与布局
```

理由:权限是所有页面的前提;进销存的商品/库存/单据数据将来被财务、生产复用,需先稳定。

## 8. 不做的事(YAGNI)

- 不引入大型中后台模板(vue-vben-admin 等),自建最小干净框架。
- 不做财务、生产模块(留路由/接口扩展位)。
- 不做工艺路线、产能排程、总账凭证、成本核算等重模块。
- 不做定制化视觉设计。
